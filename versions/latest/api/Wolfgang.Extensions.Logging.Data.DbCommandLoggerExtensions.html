Remarks

Two parameter-supplying styles are provided:

  • An explicit IReadOnlyDictionary<TKey, TValue> of parameter name to value — the lowest-overhead path.
  • A Dapper-style anonymous object (e.g. new { id = 1, name = "abc" }) — its public readable properties are reflected into a dictionary and then logged via the dictionary path. Per-type property accessors are cached so the reflection cost is paid once per parameter type, not once per call.

Parameter-name matching for excludedParameterNames is case-insensitive and prefix-tolerant: @name, :name, ?name, and name are all treated as the same parameter, so a connection's ADO.NET prefix convention does not have to be known by the caller. The value of an excluded parameter is replaced with *** in the logged output; the parameter name itself is preserved so the shape of the command is still visible.

Parameters

logger ILogger

The logger to write to.

commandText string

The SQL command text.

parameters IReadOnlyDictionary<string, object>

The command parameters, keyed by name.

Examples

logger.LogCommandText("SELECT * FROM Users WHERE Id = @id", p, new[] { "password" });
