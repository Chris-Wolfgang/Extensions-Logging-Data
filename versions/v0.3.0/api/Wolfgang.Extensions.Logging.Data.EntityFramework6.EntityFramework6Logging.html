Remarks

Add(IDbInterceptor) is process-global and not idempotent — adding the same interceptor twice doubles every log entry. To make the common "call it once per logger" pattern safe, this type keeps a static record of the loggers it has already registered and treats a repeat registration of the same logger instance as a no-op.

Methods

AddLoggingInterceptors(ILogger, IEnumerable<string>?, LogLevel)

Registers command, connection, and transaction logging interceptors for the given logger into the EF6 interception pipeline. Safe to call more than once with the same logger instance — repeat calls are a no-op.
